# Security Policy

## Supported Versions

At this stage, security fixes are provided for the latest code on the `main` branch.

## Reporting a Vulnerability

If you discover a security issue, please report it privately and do not open a public issue.

- Preferred channel: GitHub private vulnerability report (Security tab)
- Alternative: open an issue with minimal details and request a private contact path

Please include:

- A clear description of the issue
- Steps to reproduce
- Affected endpoints/files
- Potential impact
- Suggested fix (optional)

## Response Process

- We will acknowledge valid reports as soon as possible.
- We will triage severity and work on a fix.
- We will publish a patch and disclosure note after remediation when appropriate.
